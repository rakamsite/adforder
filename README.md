# Filter Inquiry Portal

## معرفی افزونه

**Filter Inquiry Portal** یک افزونه اختصاصی وردپرس برای وب‌سایت فارسی و راست‌به‌چپ است. هدف نهایی افزونه، فراهم‌کردن یک پورتال استعلام فیلتر برای مشتریان، همراه با ورود موبایلی، داشبورد کاربری، جستجوی محصول، ثبت درخواست و مدیریت اداری درخواست‌ها است.

## فاز فعلی: Phase 0

در این فاز فقط زیرساخت تمیز و قابل‌نگهداری افزونه ساخته شده است. هیچ قابلیت عملیاتی مربوط به ورود، OTP، جستجوی WooCommerce، ثبت درخواست، ارسال SMS یا مدیریت درخواست‌ها پیاده‌سازی نشده است.

## قابلیت‌های برنامه‌ریزی‌شده

- ورود و ثبت‌نام با شماره موبایل و OTP.
- تکمیل و ویرایش پروفایل مشتری.
- داشبورد مشتری برای مشاهده و ثبت استعلام‌ها.
- جستجوی محصولات WooCommerce در صورت فعال‌بودن WooCommerce.
- ثبت درخواست استعلام فیلتر.
- ارسال اعلان پیامکی از طریق Melipayamak.
- ثبت لاگ پیامک‌ها.
- پنل مدیریت برای مشاهده و رسیدگی به درخواست‌ها.
- تنظیمات افزونه در بخش مدیریت وردپرس.

## یادداشت‌های توسعه

- تمام توابع، هوک‌ها، گزینه‌ها، کلیدهای متا، اکشن‌ها، nonceها و نام‌های مرتبط با دیتابیس باید با پیشوند `fip_` ساخته شوند.
- تمام کلاس‌ها باید با پیشوند `FIP_` ساخته شوند.
- فایل اصلی افزونه فقط bootstrap را انجام می‌دهد و خروجی مستقیم ندارد.
- کلاس `FIP_Plugin` به‌صورت Singleton پیاده‌سازی شده است.
- ماژول‌ها در پوشه `includes/` نگهداری می‌شوند تا توسعه فازهای بعدی ساده‌تر باشد.
- فایل‌های asset در فاز ۰ فقط register می‌شوند و به‌صورت سراسری enqueue نمی‌شوند.
- سازگاری هدف: PHP 7.4+ و WordPress 6.x+.
- افزونه نباید به فعال‌بودن WooCommerce وابسته باشد.

## یادداشت‌های امنیتی

- تمام فایل‌های PHP دارای محافظ `ABSPATH` یا محافظ مناسب uninstall هستند.
- در فاز ۰ هیچ فرم، nonce عملیاتی، پردازش درخواست کاربر یا عملیات دیتابیس سفارشی وجود ندارد.
- در فازهای بعدی باید اعتبارسنجی، sanitize، escape، nonce و بررسی capability برای تمام ورودی‌ها و عملیات‌ها انجام شود.
- در `uninstall.php` فعلاً هیچ داده‌ای حذف نمی‌شود تا سیاست نگهداری داده در فازهای بعدی مشخص شود.

## نصب برای توسعه محلی

1. محتوای ریشه همین مخزن را به‌عنوان پوشه افزونه داخل مسیر `wp-content/plugins/filter-inquiry-portal/` وردپرس قرار دهید.
2. وارد پیشخوان وردپرس شوید.
3. از مسیر **Plugins** افزونه **Filter Inquiry Portal** را فعال کنید.
4. پس از فعال‌سازی، گزینه `fip_version` با نسخه فعلی افزونه ذخیره می‌شود.
5. برای اطمینان، افزونه را یک بار غیرفعال و دوباره فعال کنید و نبود خطای fatal را بررسی کنید.

- تولید، ارسال یا اعتبارسنجی OTP.
- اتصال به Melipayamak یا ارسال SMS.
- جستجوی محصولات WooCommerce.
- ثبت درخواست استعلام.
- داشبورد واقعی کاربر.

- ذخیره‌سازی رکوردهای درخواست یا لاگ پیامک.
- فرم‌های عملیاتی و پردازش داده‌های کاربر.
